# プロダクト概要

kb-bot は、チームの質問に **Slack / Discord** 上で自然言語で答えるナレッジ Bot。
回答の根拠を **2 つのソース**から同時に引く点が核心：チームが書いた **Markdown ドキュメント**と、
**実際のソースコード**。回答には必ず参照元（ドキュメント／ファイル・行番号）を添える。

## コア能力

- **質問の種類でルーティング**：手順・規則・用語定義は Markdown ドキュメント（R2/S3 の全文検索）、
  挙動・仕様・「なぜ」はライブの GitHub コードを読む。曖昧なら両方を読んで突き合わせる。
- **コードを信頼源（source of truth）として扱う**：ドキュメントは古くなりうるが、コードは GitHub から
  ライブで読むため常に最新。両者が食い違う場合はコードを優先するよう指示している。
- **低コスト運用**：埋め込みもベクトル DB も使わない。回答キャッシュ（SQLite 完全一致）、
  FTS5/BM25 全文検索（埋め込み API コストゼロ）、プロンプトキャッシュ、モデルティアリングで節約する。
- **マルチ LLM プロバイダ**：Anthropic / Gemini / OpenAI を中立インターフェースで切り替え可能。
- **ナレッジのライフサイクル**：解決済み Issue から知識を「育て」（`kb:issues`）、陳腐化した知識を
  「退役させる」（`kb:prune`）バッチ群を備える（任意・Bot 本体とは独立）。

## 主な利用シナリオ

- チームメンバーが Slack/Discord で「API トークンの発行方法は？」「リトライ上限とその設定場所は？」と聞く。
- モノレポを含む GitHub リポジトリの仕様・挙動を、実コードを引用しながら説明させる。
- 手書きランブックやエクスポートした Wiki を Markdown として取り込み、横断検索の対象にする。

## 価値提案

ドキュメントとコードは別の場所にあり乖離していく。kb-bot は **両方**を根拠にし、
**コードを信頼源**としつつ常に出典を示すことで、安く・信頼できる回答を返す。
ベクトル DB レスで運用が軽いことと、出典付きで答えることが差別化要因。
