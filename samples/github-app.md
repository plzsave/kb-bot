# GitHub 管理アプリの使い方

このアプリは GitHub リポジトリで Markdown ナレッジを管理し、レビュー済みの内容を配信します。

## アプリのインストール

組織の設定から GitHub App をインストールし、対象リポジトリを選択します。インストール後、Webhook 経由で push を検知します。

## ナレッジの追加

`docs/` 配下に Markdown ファイルを追加して Pull Request を作成します。
レビューが承認されてマージされると、内容が自動的にナレッジベースへ取り込まれます。

## レビューのフロー

Pull Request を作成すると AI レビューが自動でコメントします。指摘を反映してから人間のレビュアーが承認します。
緊急時はラベル `skip-review` を付けると AI レビューをスキップできます。

## 同期のタイミング

マージ後、数分以内にナレッジベースへ反映されます。すぐ反映したい場合は Actions の「Sync」ワークフローを手動実行してください。
