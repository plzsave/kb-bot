# デプロイ手順

社内アプリのデプロイに関するナレッジです。

## デプロイ先

本番環境は AWS の ECS Fargate で稼働しています。ステージング環境は同一クラスタ内の別サービスとして分離しています。
デプロイ先のリージョンは ap-northeast-1（東京）です。

## デプロイ方法

`main` ブランチへマージすると GitHub Actions が起動し、コンテナイメージをビルドして ECR に push、その後 Fargate サービスをローリング更新します。
手動デプロイが必要な場合は Actions の「Deploy」ワークフローを workflow_dispatch から実行してください。

## ロールバック

直前のタスク定義リビジョンに戻すことでロールバックできます。ECS コンソールのサービス更新から旧リビジョンを選択するか、`scripts/rollback.sh` を実行します。
